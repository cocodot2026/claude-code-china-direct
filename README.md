## 先给结论

| 你的情况 | 推荐方式 |
|---|---|
| 有海外卡 + 稳定梯子,追求绝对原生 | **官方直连**(最正,但要扛付款+封号风险) |
| 没海外卡 / 不想折腾梯子 / 要支付宝付 | **API 中转**(改两个环境变量就能用,关键是挑不降智的) |
| 自己有上游 Key、想完全自主、懂运维 | **自建聚合**(最自由,但要自己搞服务器和上游额度) |

Claude Code 本质是个读 `ANTHROPIC_BASE_URL` 的 CLI,**把它指向哪,就用谁的模型**。
所以三种方式的差别只在:**模型从哪来、钱怎么付、稳不稳、会不会被降智**。

## 三种方式对比

| 维度 | ① 官方直连 | ② API 中转 | ③ 自建聚合 |
|---|---|---|---|
| 付款 | 海外信用卡 | **支付宝/人民币** | 看你上游 Key 怎么来 |
| 梯子 | 必须 | 通常不用 | 看部署位置 |
| 封号风险 | 有(Anthropic 风控) | 转移给中转方 | 看上游 |
| 降智风险 | 无 | **有(命门,要验)** | 看你接的上游 |
| 上手成本 | 中(要卡+梯子) | 低(改 2 个变量) | 高(要运维) |
| 自主度 | 高 | 中 | 最高 |

## 方式一:官方直连(梯子 + 海外卡)

直接用 Anthropic 官方账号和 API。

- **优点**:最原生、零中间商、绝不降智。
- **缺点**:
  1. **付款**:要一张能过 Anthropic 风控的海外信用卡(国内卡基本被拒)。
  2. **封号**:支付卡段 + 登录 IP + 主体不一致,容易触发风控封号。
  3. **梯子**:全程要稳定的网络环境。
- **适合**:已经有海外卡和稳定环境、追求绝对原生的人。

> 付款这块如果卡住,可以用美国卡段虚拟卡绑 Anthropic;挑卡的坑另见
> [出海/AI 订阅支付指南](https://github.com/cocodot2026/pay-for-overseas-ads-from-china)。

## 方式二:API 中转(改两个环境变量)

把 Claude Code 指向一个 OpenAI/Anthropic 兼容的中转端点。**这是没有海外卡的人最省事
的方式**——不用梯子、支付宝付、按量计费。配置就两步:

```bash
# 1. 指向中转的 Anthropic 兼容端点 + 你的 Key
export ANTHROPIC_BASE_URL="https://<你的中转>/api/ai"
export ANTHROPIC_AUTH_TOKEN="你的_KEY"

# 2.(可选)指定模型,大小模型分开配更省钱
export ANTHROPIC_MODEL="<大模型代号>"
export ANTHROPIC_SMALL_FAST_MODEL="<小模型代号>"

# 然后正常跑
claude
```

- **优点**:上手最快、支付宝付、通常不用梯子、按 token 计费。
- **命门**:**降智**。差的中转会偷偷把你请求的 Opus 换成便宜小模型、或高峰期限流截断,
  你付 Opus 的钱拿 Haiku 的活。**所以选中转 = 选一家不降智的**,而且要能**自己验证**
  (见下一节)。
- **适合**:没海外卡、不想折腾、要快速用起来的开发者。

> **拿 cocodot 举例(作者自家,供参考格式)**:端点 `https://cocodot.co/api/ai`,
> 环境变量 `ANTHROPIC_BASE_URL` + `ANTHROPIC_AUTH_TOKEN`,模型代号
> `mco-6`=Opus 4.8 / `mcs-5`=Sonnet 4.6 / `mch-1`=Haiku 4.5,官网从 9 折起、按量计费、
> 支付宝充值。**别因为是我就加分——拿下面的验降智工具去测任何一家。**

## 方式三:自建聚合(one-api / new-api)

自己部署 [one-api](https://github.com/songquanpeng/one-api) / new-api 这类聚合网关,
填入你自己的上游 Key,再把 Claude Code 指向它。

- **优点**:完全自主、多上游可切换、可自己做用量统计。
- **缺点**:你得**自己搞到上游额度**(还是回到付款问题)、自己扛服务器和运维、上游降不
  降智还是得自己盯。
- **适合**:已经有上游 Key、懂运维、要完全掌控的人。

## 核心:怎么验「有没有被降智」

**这是选中转最关键、也最少人讲的一步。** 别信任何中转的口头承诺("我们官转不降智"),
**自己测**。有个开源 CLI 工具 [LLMprobe](https://github.com/cocodot2026/LLMprobe) 就是干
这个的:

- 对任意 OpenAI 兼容端点跑 6 类探针:**模型身份、能力(用另一个模型当裁判打分)、
  延迟、上下文长度、限流、一致性**。
- **Key 不落库**、开源可审计,你自己跑自己看结果。
- 输出一个综合分,谁降智一测就露馅。

```bash
# 对任意中转端点自测(示例)
pip install openai
python llmprobe.py --base-url https://<你的中转>/api/ai/v1 --api-key <你的KEY> --model <模型>
```

> 作者用它实测过 cocodot = **87/100**(真 Claude、零降智);但**重点不是这个分,是这个
> 方法**——你换任何一家,都用同一把尺子量,谁虚谁实自己看。

## 怎么挑一家中转(选型 6 条)

不管你最后选谁(包括不选 cocodot),拿这 6 条去量:

1. **不降智可验证**:能不能用上面的开源工具测出真模型?还是只会嘴上保证?
2. **费率透明**:倍率/按量怎么算,有没有藏起来的加价。
3. **模型齐不齐**:Claude / GPT / Gemini / DeepSeek 一个 Key 能不能都调。
4. **付款方式**:支持不支持支付宝/人民币(没海外卡的关键)。
5. **稳定性/限速**:高峰会不会截断、限流,有没有 SLA 说法。
6. **接入兼容**:是不是标准 OpenAI/Anthropic 兼容端点(能不能无痛接 Claude Code /
   Cursor / Cline / Cherry Studio)。

## FAQ

**Claude Code 能在国内用吗?** 能。它就是读 `ANTHROPIC_BASE_URL` 的 CLI,指向官方走方式
一,指向中转走方式二,指向自建走方式三。

**没有海外信用卡怎么用 Claude Code?** 走方式二(API 中转,支付宝付)最省事;或用美国卡段
虚拟卡绑官方走方式一。

**中转会不会把我的 Opus 偷换成小模型?** 差的会。所以别信口头保证,用
[LLMprobe](https://github.com/cocodot2026/LLMprobe) 自己测——这是本指南最想让你记住的一件事。

**中转安全吗?我的 Key/代码会不会被看?** 你的请求会过中转服务器,选**开源可审计、明确不
落库**的;敏感项目建议自建(方式三)。

**大小模型怎么配更省钱?** `ANTHROPIC_MODEL` 配大模型(难任务)、`ANTHROPIC_SMALL_FAST_MODEL`
配小模型(补全/简单活),Claude Code 会自动分流。
